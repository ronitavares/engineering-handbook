# Knowledge Consolidation Checklist

Use este checklist antes de considerar uma feature concluída.

---

## 1. Artefatos transitórios

- [ ] Feature Design revisada.
- [ ] Execution Plan revisado.
- [ ] Scratchpads removidos.
- [ ] Prompts/conversas não foram versionados como fonte permanente.
- [ ] Diagramas temporários removidos ou promovidos.

---

## 2. Decisões permanentes

- [ ] Houve decisão arquitetural permanente?
  - [ ] Não.
  - [ ] Sim, ADR criado.

---

## 3. Guidelines

- [ ] Surgiu padrão reutilizável?
  - [ ] Não.
  - [ ] Sim, Guideline criada/atualizada.

---

## 4. Produto

- [ ] Houve mudança de regra de negócio?
  - [ ] Não.
  - [ ] Sim, PRD atualizado.

---

## 5. Contratos

- [ ] Houve mudança em API, evento ou schema?
  - [ ] Não.
  - [ ] Sim, contrato atualizado.
  - [ ] Versionamento validado.

---

## 6. Testes e validação

- [ ] Testes automatizados passam.
- [ ] Critérios de aceite validados.
- [ ] Regressão relevante considerada.
- [ ] Observabilidade validada, se aplicável.
- [ ] Segurança revisada, se aplicável.

---

## 7. Encerramento

- [ ] A feature não deixou conhecimento permanente apenas em chat/prompt.
- [ ] O repositório não recebeu documentação duplicada do código.
- [ ] O que ficou no repositório é durável e possui owner.
- [ ] A entrega pode ser considerada concluída.
